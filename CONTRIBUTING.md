# دليل المساهمة في مشروع Doctor Syria

## كيفية المساهمة

نحن نرحب بمساهماتكم في تطوير منصة Doctor Syria! هذا الدليل يوضح الخطوات الأساسية للمساهمة في المشروع.

### 1. إعداد بيئة التطوير

1. استنسخ المشروع:
```bash
git clone https://github.com/khaled8809/doctor-syria-v2.git
cd doctor-syria-v2
```

2. قم بإنشاء وتفعيل البيئة الافتراضية:
```bash
python -m venv venv
source venv/bin/activate  # على Windows: venv\Scripts\activate
```

3. ثبّت المتطلبات:
```bash
pip install -r requirements.txt
```

### 2. العمل على ميزة جديدة

1. قم بإنشاء فرع جديد:
```bash
git checkout -b feature/اسم-الميزة
```

2. اعمل على التغييرات المطلوبة

3. أضف واعتمد التغييرات:
```bash
git add .
git commit -m "وصف التغييرات"
```

4. ارفع التغييرات إلى GitHub:
```bash
git push origin feature/اسم-الميزة
```

5. أنشئ Pull Request على GitHub

### 3. معايير الكود

- اتبع معايير PEP 8 لكود Python
- اكتب تعليقات توثيقية للدوال والفئات
- أضف اختبارات للكود الجديد
- تأكد من أن جميع الاختبارات تنجح قبل إرسال Pull Request

### 4. إرشادات Pull Request

- استخدم عنواناً وصفياً
- اشرح التغييرات بالتفصيل
- أضف صور للتغييرات المرئية (إن وجدت)
- اربط PR بـ Issue ذات صلة (إن وجد)

### 5. الإبلاغ عن المشاكل

- استخدم نظام Issues على GitHub
- اتبع قالب الإبلاغ عن المشاكل
- قدم أكبر قدر ممكن من التفاصيل
- أضف خطوات إعادة إنتاج المشكلة

### 6. أفضل الممارسات

- حافظ على تحديث فرعك مع main
- اجعل الـ commits صغيرة ومركزة
- اختبر التغييرات قبل الـ push
- وثق التغييرات في README إذا لزم الأمر

### 7. التواصل

- استخدم Discussions على GitHub للأسئلة العامة
- استخدم Issues للمشاكل والاقتراحات
- كن محترماً ومهنياً في جميع التفاعلات

شكراً لمساهمتك في تطوير Doctor Syria! 🌟
